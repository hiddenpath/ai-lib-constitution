## 1. 双维度矩阵定义

| | **To B（企业/开发者）** | **To C（个人用户）** |
|---|---|---|
| **Prism（P层·平台侧）** | ① Prism Enterprise | ② Prism |
| **Vela（A层·用户端）** | ③ Vela Pro | ④ Vela |

四个象限不是四个独立产品，而是**同一技术栈在不同市场的不同交付形态**。

## 2. 逐象限拆解

### ① Prism Enterprise（Prism × To B）

**目标客户**：中大型企业 IT/SRE 团队、合规部门、需要多 Provider 统一接入的 SaaS 公司

**核心价值**：不用自己建 AI 网关，Prism 全托管——Key 池、路由、计费、合规、审计全包

**功能**：

- 多租户隔离（Phase 2+）
- SSO/SAML（Phase 2+）
- 审计报表（Phase 2+）
- 策略推送：管理员定义路由策略，推到全公司客户端（Phase 2+）
- 私有部署 Docker（Phase 2+）
- SLA 99.9%（Phase 3+）

**定价**：年费（按团队规模）+ 按量加价

**需厘清的问题**：

**P2B-1：Enterprise 与 Prism② 共享代码库？**
- 选项 A：同一仓库 + feature flag（`--features enterprise`）
- 选项 B：独立仓库，共享核心 crate
- **判断**：选项 A。Rust feature flag 天然支持，避免"免费版修了 bug 企业版忘了"。

**P2B-2：私有部署 License 模式？**
- Phase 2：Docker 镜像 + 在线激活（最简）
- Phase 3：视大客户需求考虑源码授权（NDA）

**P2B-3：审计报表数据范围？**
- Prism 只存 ExecutionMetadata（provider/model/latency/usage/cost），不存对话内容
- 企业审计需要"用量 + 成本 + Provider 健康"——当前架构天然满足

---

### ② Prism（Prism × To C）

**目标客户**：个人开发者、独立开发者、小团队、技术爱好者

**核心价值**：一个 API Key 访问所有 Provider，Prism 帮你路由、降级、计费

**功能**：

- OpenAI 兼容 API + 流式 SSE（Phase 1）
- Key 池托管：轮换/冷却/熔断/恢复（Phase 1）
- 基础路由：round-robin + fallback（Phase 1）
- 5 个 P0 Provider（Phase 1）
- 智能路由 auto 模式（Phase 2）
- BYOK（Phase 2）
- 用户 Provider 偏好设置（Phase 2）
- 多模态扩展（Phase 3+）

**定价**：

| Phase | 模式 | 说明 |
|-------|------|------|
| 1 | 免费 | 闭环验证 |
| 2 | 按量 + margin | Provider 成本价 + 平台费 |
| 2 | 订阅套餐 | 月费含配额，超出按量 |

**需厘清的问题**：

**P2C-1：免费 vs 收费的边界？**
- 选项 A：纯按量加价（最简，margin 受竞品压制）
- 选项 B：基础免费 + 增值付费（与 Enterprise 重叠）
- 选项 C：免费配额 + 超出付费（零门槛 + 自然转化）
- **判断**：Phase 2 用选项 A（简），Phase 3 迁移到选项 C（优）。

**P2C-2：差异化如何传递给 To C 用户？**
- To C 用户不买"架构优势"，买**结果优势**
- 传播路径：
  - 协议运行时 → "切换 Provider 不换代码"（省心）
  - Key 池托管 → "不用担心 Key 被封"（稳定）
  - 架构级隐私 → "我们存不了你的对话"（信任）
  - 中文原生 → "DeepSeek/Qwen 直连不绕美国"（快 + 便宜）

**P2C-3：margin 空间？**
- 不是"OpenRouter + 5%"，而是"总拥有成本更低"
- margin 来自：智能路由省的钱 > 平台费；Key 池托管省的运维；中文市场的本土化溢价

---

### ③ Vela Pro（Vela × To B）

**目标客户**：需要在产品中嵌入 AI 能力的开发者 / SaaS 公司

**核心价值**：一行代码嵌入 Prism 的路由能力到你的应用里

**功能**：

- Prism SDK（`@ailib/prism-sdk`）— 开源免费（Phase 1）
- WASM 路由嵌入 — 浏览器端协议执行 + 路由决策（Phase 2）
- 团队历史同步 — E2E 加密，零知识（Phase 2+）
- 合规模板 — 医疗/金融/法律预置路由策略（Phase 2+）
- CLI 工具（Phase 2）

**定价**：

| 形态 | 模式 |
|------|------|
| SDK | 开源免费（A 带） |
| WASM 路由模块 | 基础开源 / 高级付费 |
| 团队同步 | 按人头/月 |
| 合规模板 | 按行业包购买（C 带） |

**需厘清的问题**：

**V2B-1：WASM 路由嵌入是核心差异化还是锦上添花？**
- 多数开发者的 AI 调用在服务器端，客户端 WASM 路由需求待验证
- **判断**：Phase 1 不以 WASM 路由为卖点。Vela Pro = Prism SDK（纯 API 调用）。WASM 路由 Phase 2 作为"隐私 + 离线"增强功能。

**V2B-2：合规模板包含什么？**
- 选项 A：纯路由策略（YAML 配置）
- 选项 B：路由策略 + 审计钩子
- 选项 C：完整行业 SDK（路由 + 审计 + 脱敏 + 数据格式适配）
- **判断**：Phase 2 做选项 A，Phase 3 做选项 B，选项 C 等行业合作方

**V2B-3：Vela Pro 与 Prism Enterprise 的关系？**
- **判断**：Phase 2 卖 **Prism + Vela Pro 捆绑套餐**（网关 + SDK + 审计 + 支持）。单独 Vela Pro 不含 Prism 不完整。Phase 3 再拆卖。

---

### ④ Vela（Vela × To C）

**目标客户**：日常使用 AI 的个人用户

**核心价值**：帮用户选最好的模型，对话数据留在本地

**功能**：

- 聊天 UI + 多模型切换（Phase 1）
- 本地历史 IndexedDB（Phase 1）
- 智能推荐："这个问题用 DeepSeek 最划算"（Phase 2）
- 多设备 E2E 加密同步（Phase 2+，付费）
- 模型对比并排显示（Phase 2）
- 一键导出 Markdown/PDF（Phase 1）

**定价**：

| 功能 | 免费 | 付费 |
|------|------|------|
| 基础聊天 + 本地历史 | ✅ | |
| 多设备同步 | | ✅ 月费 |
| 高级路由（auto） | | ✅ |
| 模型对比 | ✅（2 模型） | ✅（无限） |

**需厘清的问题**：

**V2C-1：Vela To C 在 Phase 1 的优先级？**
- Prism Phase 1 核心闭环是**开发者用 API 调用模型**
- **判断**：Phase 1 Vela 做到"可演示的最小聊天界面"即可（验证 Prism API），不做消费者级体验。Phase 2 再升级。

**V2C-2：Vela vs Poe 的生存空间？**
- Poe 多模态体验极好，Vela 差异化在**隐私**和**透明**
- 短期受众：中国开发者、隐私敏感人群、成本透明用户——小众但高忠诚度
- **Phase 策略**：Phase 1 不和 Poe 竞争消费者市场，Phase 2 以隐私 + 透明切入细分

## 3. 跨象限能力归属

| 能力 | ①Enterprise | ②Prism | ③Vela Pro | ④Vela | 归属 |
|------|:---:|:---:|:---:|:---:|------|
| E 层协议执行 | 共享 | 共享 | 共享 | 共享 | A 带 |
| Key 池轮换 | ✅ | ✅ | — | — | P 层 Prism 独占 |
| 基础路由 | ✅ | ✅ | — | — | P 层 Prism 独占 |
| 智能路由 | ✅ | ✅(P2) | — | — | P 层 Prism 独占 |
| 客户端路由(WASM) | — | — | ✅ | ✅(P2) | A 层 Vela 独占 |
| 用量记录 | ✅ | ✅ | — | — | P 层 |
| 审计报表 | ✅ | — | — | — | C 带 Enterprise 独占 |
| SSO/SAML | ✅ | — | — | — | C 带 Enterprise 独占 |
| 策略推送 | ✅(发) | — | ✅(收) | — | C 带 |
| BYOK | ✅ | ✅(P2) | — | — | P 层 |
| 本地历史 | — | — | ✅ | ✅ | A 层 Vela 独占 |
| E2E 加密同步 | ✅(密文存储) | ✅(P2) | ✅ | ✅(P2付费) | 跨层协作 |
| 合规模板 | ✅ | — | ✅ | — | C 带 To B 独占 |
| 聊天 UI | — | — | — | ✅ | A 层 Vela To C |

**三个关键洞察**：

1. **Prism 和 Vela 的能力几乎不重叠**——E/P 分离的自然结果
2. **唯一跨层协作点是 E2E 加密同步**——Prism 存密文，Vela 管密钥，是 A/P 合同边界和 To B/C 收费分界
3. **合规模板是 To B 独占价值**——验证了矩阵设计的合理性

## 4. 阶段路线图

### Phase 1（3 周）：验证闭环

| 象限 | 交付物 | 投入 |
|------|--------|------|
| ② Prism | 核心 API + Key 池 + 基础路由 + 5 Provider | 80% |
| ④ Vela | 最小聊天 UI（验证 Prism API） | 20% |
| ① Enterprise | — | 不做 |
| ③ Vela Pro | Prism SDK（开源客户端库） | 随 Prism 产出 |

**完成标准**：`api.prism.ailib.info` 外部可用，5 Provider 全通，Key 池 + 降级在线

### Phase 2（6-8 周）：规模化 + 初步商业化

| 象限 | 交付物 | 说明 |
|------|--------|------|
| ② Prism | 智能路由 + BYOK + 用户偏好 + 计费 | 核心产品升级 |
| ① Enterprise | 多租户 + SSO + 审计 + 私有部署 | To B 首版 |
| ③ Vela Pro | WASM 路由 + 团队同步 + 合规模板(轻) | To B 嵌入能力 |
| ④ Vela | 智能推荐 + 模型对比 + E2E 同步(付费) | To C 体验升级 |

**完成标准**：首个付费用户，Prism 按量计费在线，Enterprise 私有部署可交付

### Phase 3（3-6 月）：商业化 + 生态

| 象限 | 交付物 | 说明 |
|------|--------|------|
| ② Prism | 多模态 + 本土支付 + 订阅制 | 收入模式成熟 |
| ① Enterprise | SLA 99.9% + 合规模板(重) + 专属支持 | 高客单价 |
| ③ Vela Pro | 合规模板(审计钩子) + 自托管 | 垂直行业渗透 |
| ④ Vela | 多设备漫游 + 高级分析 | To C 付费转化 |

## 6. 与竞品矩阵对比

| | **To B** | **To C** |
|---|---|---|
| **平台侧** | Prism Enterprise vs OpenRouter | Prism vs OpenRouter(免费层) |
| **客户端** | Vela Pro vs LangChain/LlamaIndex | Vela vs Poe/ChatGPT |

**关键发现**：

1. **OpenRouter 只占一个象限**（平台侧 To B），没有客户端产品
2. **Poe 只占一个象限**（客户端 To C），没有平台侧独立产品
3. **Prism/Vela 覆盖全部四个象限**——这是架构优势的市场体现
4. **LangChain/LlamaIndex 在 Vela Pro 象限**，但它们是编排框架不是协议运行时——不同生态位

## 7. 三带模型与矩阵的对应

| | A 带（永久开源） | B 带（开源非 SLA） | C 带（可闭源） |
|---|---|---|---|
| **① Enterprise** | E 层 + Prism SDK | Vela Pro E2E 客户端 | 路由引擎 + 审计 + SSO + 合规模板 |
| **② Prism** | E 层 + Prism SDK | — | Key 池 + 计费 + Admin API |
| **③ Vela Pro** | Vela 客户端核心 + WASM 基础 | 团队同步客户端 | 高级 WASM 路由 + 合规模板 |
| **④ Vela** | Vela 聊天 UI + 本地历史 | — | E2E 同步服务 + 高级路由 |

**洞察**：每个象限都有 A 带基础（开源引流）和 C 带增值（商业收费），B 带只在特定象限出现（Enterprise 和 Vela Pro 的"可自托管"组件）。
