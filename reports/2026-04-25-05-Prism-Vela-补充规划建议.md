## 0. 为什么报告 1-4 缺了 Prism/Vela

4 份报告的审阅对象是《ai-lib开源分界产品规划和商业模式》——这是一份**通用 AI 生态产品构想**，写于 Prism/Vela 决策之前。报告人基于 MEMORY + YAML 做了严谨校准，但 Prism/Vela 的核心决策（双品牌、闭源 P 层、上下文存储策略）发生在 4 月 20 日，而报告框架沿用的是更早的产品矩阵术语（AI-Capability Router / Pack / Mesh / Console 等）。

**核心矛盾**：报告中的"AI-Capability Router"其实就是 Prism；"Enterprise Console"是 Prism Phase 3+；"Zero-Config Hybrid Router"是 Vela + WASM 路径。但报告把它们当独立构想评估，没有识别为 **同一个产品（Prism/Vela）的不同阶段**。

## 2. Prism 在三带模型中的定位

### 2.1 明确归属

| Prism 模块 | A 带（永久开源） | B 带（开源非 SLA） | C 带（可闭源） |
|-----------|:---:|:---:|:---:|
| Prism Core（路由调度 + Key 池 + 计费 + 降级） | | | ✅ |
| Prism Admin API（用户/Key/用量管理） | | | ✅ |
| Prism Pack 分发服务（签名 + 注册表） | | | ✅ |
| Prism SDK 客户端库（调用 Prism 的轻量 SDK） | ✅ | | |
| Prism 协议适配层（ai-lib-core E 层调用） | ✅ | | |

**原则**：
- **C 带核心**：路由引擎、Key 池调度、定价引擎、运营面板 = Prism 的商业护城河
- **A 带入口**：用户接入 Prism 的客户端 SDK（`@ailib/prism-sdk`）应开源，降低接入门槛
- **E 层复用**：Prism 直接依赖 ai-lib-core（A 带），但不暴露 P 层实现

### 2.2 与报告 3 的衔接

报告 3 的三带模型没有 Prism 位置。修正后：

> **C 带定义（补充）**：Prism 平台闭源模块（路由引擎、Key 池、计费、管理 API、Pack 签名分发）是 C 带的核心产品。这不是"可能做"的未来构想，而是**已决的产品名和定位**。Prism 的存在让 C 带不再是抽象概念，而是有名字、有架构、有 Phase 1 计划的具体交付物。

## 3. Vela 在三带模型中的定位

| Vela 模块 | A 带 | B 带 | C 带 |
|-----------|:---:|:---:|:---:|
| Vela Web 客户端（聊天 UI + 导航） | ✅ | | |
| Vela 本地历史（IndexedDB） | ✅ | | |
| Vela WASM 路由决策（客户端 side） | ✅ | | |
| Vela E2E 加密同步客户端 | | ✅ | |
| Vela 云端历史同步服务 | | | ✅ |

**原则**：
- **A 带主体**：Vela 的客户端核心开源（聊天 UI、本地历史、WASM 路由决策）
- **B 带**：E2E 加密同步的客户端实现可开源，但声明"非 SLA / 自承载"
- **C 带**：Prism 侧的密文存储 + 同步服务 = 付费增值（Phase 2）

### 3.1 Vela 的 WASM 路由能力（与报告 1 的衔接）

报告 1 指出"协议进 WASM、薄代理"是强一致项。Vela 正是这个路径：

```
Vela（客户端）→ WASM 路由决策 → Prism（P 层）→ Provider
                     ↓
              客户端可离线决策
              "用 DeepSeek 最便宜"
              "用 Claude 质量最高"
              "本地模型最隐私"
```

这与报告 2 中"Zero-Config Hybrid Router"构想一致，但应归入 **Vela 产品线**而非独立构想。

## 4. 上下文存储策略与报告的衔接

### 4.1 已决策略

| Phase | 上下文策略 | Prism 存什么 | Vela 存什么 | 收费 |

### 4.2 与报告 2 的修正

报告 2 评估"托管透明协议 Proxy"时说"仅当有明确商业模式与运维时再立项"——但 **Prism Phase 1 已经立项**（有 3 周执行计划）。应修正为：

> Prism Phase 1 已决策立项。评估重点从"是否做"转为"Phase 1 范围是否合理"。Phase 1 的纯元数据 + 纯客户端策略回避了隐私合规风险，是保守且正确的起点。
